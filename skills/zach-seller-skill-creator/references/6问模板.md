# 6 维映射参考：阶段 0 采集完后，如何落到 SKILL.md

这份文档是 `zach-seller-skill-creator` 的**内部参考材料**，不是给用户填写的大表单。

用途：
- 帮 Claude 把阶段 0 收集到的 6 维答案映射到 `SKILL.md`
- 作为生成初稿时的字段提取参考
- 提供一个卖家场景示例，方便校验答案粒度够不够

不要把这份文档整段发给用户填写。阶段 0 的正式问法以 `references/6问引导式流程.md` 为准。

---

## 第一部分：6 维到 SKILL.md 的映射

| 阶段 0 维度 | 进入 SKILL.md 的哪里 | 提取重点 |
|------------|----------------------|----------|
| Q1 业务目标 | `## 业务背景` | 业务目标、商业结果、不做的损失 |
| Q2 现有做法 | `## 当前工作流（人工版）` | 当前做法、耗时、痛点、已有工具 |
| Q3 具体步骤 | `## Skill 工作流（自动版）` | 步骤顺序、输入、输出、判断条件 |
| Q4 方法论 | `## 核心原则 / 踩坑规避` | 阈值、优先级、例外条件、禁忌 |
| Q5 调用方式 | YAML `description` + `## 触发场景` | 触发方式、真实话术、业务入口 |
| Q6 期望输出 | `## 输出规范` | 产物类型、保存位置、命名规则、完成信号 |

## 第二部分：字段提取提示

### Q1 业务目标

至少提取出：
- 这个 skill 解决什么业务问题
- 不做会损失什么
- 最好保留用户原话里的量纲：时间、钱、销量、机会

适合写进：
- `## 业务背景`
- YAML `description` 中的“做什么 + 什么时候用”

### Q2 现有做法

至少提取出：
- 当前是纯手动、半自动还是全新流程
- 现在已经在用什么工具
- 一次做多久、最烦哪一步

适合写进：
- `## 当前工作流（人工版）`
- 也可用于解释为什么这个 skill 值得做

### Q3 具体步骤

每一步尽量提取：
- 动作
- 输入
- 输出
- 判断

如果用户只给动作顺序，没有判断逻辑，不要直接写入 `SKILL.md`，先回到阶段 0 补问。

适合写进：
- `## Skill 工作流（自动版）`
- 如步骤太长，可把细节拆到 `references/` 再在正文里指路

### Q4 方法论

重点提取：
- 具体阈值
- 趋势/对比口径
- 优先级规则
- 例外排除条件

亚马逊卖家常见信号：
- `BCG 健康阈值`
- `广告占比异常`
- `新品期不纳入稳定判断`
- `节假日周单独处理`

适合写进：
- `## 核心原则 / 踩坑规避`

### Q5 调用方式

重点提取：
- 默认触发方式：一句话 / 先问参数 / 上传文件
- 最常见的业务入口：Sorftime、领星、SIF、本地表格、飞书等
- 2-3 句真实触发话术

适合写进：
- YAML `description`
- `## 触发场景`

### Q6 期望输出

重点提取：
- 产物类型
- 输出落点
- 命名规则
- 完成信号

适合写进：
- `## 输出规范`

## 第三部分：卖家场景示例

**场景**：卖家每周都要做一次 BCG ASIN 的关键词自然排名检查，现在想把这个流程做成 skill。

### Q1 业务目标

- 目标：及时发现自然排名掉出前 3 页的核心 ASIN，触发补救（补评论、调 listing、或加广告拉回流量）
- 不做的损失：自然流量是免费流量，排名掉了要拿广告钱补，一周没发现可能损失几千美金广告费 + 销量下滑

### Q2 现有做法

- 每周一早上手动做
- 打开 Sorftime，搜 BCG 这个 ASIN，看每个核心词的排名位置
- 把结果复制到 Excel，和上周数据做对比
- 每个 ASIN 约 15 分钟，12 个 ASIN 一共要 3 小时
- 最烦的是复制和对比，经常因为嫌麻烦跳过

### Q3 具体步骤

- 第 1 步：从 Sorftime 拉 BCG 的核心 ASIN 关键词数据
- 第 2 步：过滤出核心词清单
- 第 3 步：对比上一周记录；如果没有上周数据，就先记当前周基线
- 第 4 步：标记排名变化 > 10 位的异常词；如果掉出前 20 名，升级为严重
- 第 5 步：生成 Markdown 报告并保存到固定目录
- 第 6 步：把摘要发到飞书运营群

### Q4 方法论

- 排名变化 < 5 位不算异常
- 新品上架 < 30 天不纳入稳定判断
- 节假日周单独处理，不和普通周硬比
- 同一个核心词连续 2 周下滑，必须升级人工介入

### Q5 调用方式

- 触发方式：一句话直接触发
- 业务入口：Sorftime 研究 + SIF 排名诊断
- 真实话术：
  - “帮我查一下 BCG 这周的关键词排名”
  - “跑一下每周的关键词排名检查”
  - “看下哪些词掉出前 20 了”

### Q6 期望输出

- 产物类型：Markdown 报告 + 飞书消息 + 异常清单
- 落点：本地目录 + 飞书群
- 命名规则：`YYYY-MM-DD_BCG_关键词自然排名周报.md`
- 完成信号：终端回一句“已完成，异常 N 个、严重 M 个，报告已发到飞书”

## 第四部分：生成时的自检清单

开始写 `SKILL.md` 前，快速对照：

- `业务背景` 里是否写清 why，而不是只写“要做什么”
- `当前工作流（人工版）` 里是否有真实人工基线，不是空壳
- `Skill 工作流（自动版）` 里是否能看出判断逻辑
- `核心原则 / 踩坑规避` 是否至少有一条具体规则
- `触发场景` 是否写入真实话术，而不是抽象标签
- `输出规范` 是否同时写明产物形式和落点

如果以上任一项回答不上来，不要硬写，先回阶段 0 补采集。
