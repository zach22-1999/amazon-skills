# 判定标准 — 三维度信号判定 + 综合判定矩阵

## 维度一：Review 信号判定

### 指标定义

| 指标 | 计算方式 |
|------|---------|
| 功能提及率 | 功能相关评论数 / 总评论数 × 100% |
| 正面/负面比 | 功能相关正面评论数 : 功能相关负面评论数 |
| 需求信号计数 | 包含 "wish it had" / "would be nice if" / "missing" 等表达的评论数 |

### 信号判定表

| 信号强度 | 条件 | 说明 |
|---------|------|------|
| 🟢 强信号 | 功能提及率 > 10%，或需求信号 ≥ 5 条 | 消费者明确表达了对该功能的需求或好评 |
| 🟡 弱信号 | 功能提及率 3%-10%，或需求信号 1-4 条 | 有一定信号，但不够强烈 |
| 🔴 无信号 | 功能提及率 < 3%，且无需求信号 | 消费者不关心这个功能 |
| ⚠️ 负面信号 | 负面提及占功能相关评论 > 60% | 功能存在但用户不满意 |

### 采集策略

**场景 A：品类中有含该功能的产品**
→ 拉取含该功能的产品（3-5 个）的正面+负面评论
→ 统计功能提及率和情感分布

**场景 B：品类中无含该功能的产品**
→ 拉取品类头部产品（Top 3-5）的差评
→ 重点搜索"缺少该功能"的反向需求信号（wish it had / if only / needs a）
→ 这种场景下，即使需求信号少也值得关注（因为消费者不会主动在差评中要求不存在的功能）

## 维度二：关键词信号判定

### 指标定义

| 指标 | 数据源 | 说明 |
|------|--------|------|
| 周搜索量 | keyword_detail | 核心功能词的周搜索量 |
| 搜索排名 | keyword_detail | 在亚马逊全部关键词中的排名 |
| 趋势方向 | keyword_trend | 近 12 个月搜索量走势 |
| 相关延伸词数量 | keyword_extends | 与功能直接相关的延伸词有多少个 |

### 信号判定表

| 信号强度 | 条件 | 说明 |
|---------|------|------|
| 🟢 强信号 | 核心词周搜索量 > 5,000 且趋势上升，或相关延伸词 ≥ 10 个 | 消费者主动且持续搜索该功能 |
| 🟡 弱信号 | 核心词周搜索量 1,000-5,000，或趋势平稳但有延伸词 | 有一定搜索行为 |
| 🔴 无信号 | 核心词周搜索量 < 1,000 且无延伸词 | 消费者不主动搜索该功能（可能是伪需求或市场教育不足） |
| 📈 增长信号 | 搜索量季度环比增长 > 30% | 需求正在被激活（叠加到其他判定之上） |
| 未验证 | 当前环境无 Sorftime MCP，未完成关键词数据采集 | 只能作为降级版结果，不得当成完整判定 |

### 关键词查询规则

- ⛔ **至少查 3 个关键词变体**，不能只查一个词就下结论
- 如果核心词无数据，查看 Sorftime 推荐的相关词是否有线索
- "steam oven" 这类相邻品类词可作参考但不直接用于判定

### 趋势判断规则

- **上升**：近 6 个月搜索量整体走高，最近 3 个月均高于 6 个月前
- **平稳**：波动幅度 < 50%，无明确方向
- **下降**：近 6 个月搜索量整体走低
- **波动/不规则**：搜索量在多个月份降至 0，无稳定模式
- **季节性**：仅在特定月份出现搜索量

## 维度三：社区信号判定

### 指标定义

| 指标 | 数据源 | 说明 |
|------|--------|------|
| 相关讨论数 | WebSearch Reddit + Quora | 直接讨论该品类+功能的帖子/问题数量 |
| 用户态度 | 帖子内容 | 期待 / 推荐 / 质疑 / 吐槽 |
| 时间趋势 | 发布日期 | 讨论是近期增多还是冷门话题 |

### 信号判定表

| 信号强度 | 条件 | 说明 |
|---------|------|------|
| 🟢 强信号 | 相关讨论 ≥ 10 条，且多数态度正面（期待/推荐） | 消费者购买前积极讨论该功能 |
| 🟡 弱信号 | 相关讨论 3-9 条，态度混合 | 有一定关注但不热烈 |
| 🔴 无信号 | 相关讨论 < 3 条 | 消费者不讨论该功能 |
| ⚠️ 负面信号 | 讨论多但态度以质疑/吐槽为主 | "gimmick"、"不实用"等负面标签 |

### 搜索策略

1. **Reddit**：`site:reddit.com "[品类] [功能]"` + 变体搜索
2. **Quora**：`site:quora.com [品类] [功能] worth it`
3. **搜索结果为零也是有效信号** — 说明消费者不讨论该功能

## 综合判定矩阵

| 判定 | 条件 | 建议 |
|------|------|------|
| ✅ **强真需求** | 3 个维度均有正面信号（🟢 或 🟡） | 值得投入开发，功能作为核心卖点 |
| ⚠️ **弱真需求** | 2 个维度有正面信号，1 个缺失（🔴） | 可考虑，但需关注缺失维度的风险 |
| ❓ **待验证** | 仅 1 个维度有信号 | 建议进一步调研或小批量试水 |
| ❌ **伪需求** | 无正面信号，或有明显负面信号（⚠️） | 不建议投入，该功能可能是噱头 |

> 如果关键词维度是“未验证”，综合判定必须同步降低信心等级，并在风险提示里写明“当前结论不包含搜索行为验证”。

## 特殊情况处理

| 情况 | 解读 | 建议 |
|------|------|------|
| 关键词无信号 + Review/社区有强信号 | "市场教育不足"型需求 | 可作先发优势切入，但需预期市场教育成本 |
| 关键词有信号 + Review 负面 | 功能有需求但现有实现有问题 | 找到改进方向后可做 |
| 仅社区有信号 | "讨论热但购买冷"型 | 需谨慎，可能是伪需求 |
| Review 弱信号仅在高端价位 | 功能在特定价位段有需求 | 需评估自己的产品定位是否匹配 |

## 信心等级判定

| 信心等级 | 条件 |
|---------|------|
| **高** | 三个维度数据充分（Review 样本 ≥ 200、关键词 ≥ 3 个有数据、社区 ≥ 5 条讨论） |
| **中** | 两个维度数据充分，一个数据较少 |
| **低** | 数据整体不足，或存在矛盾信号 |
