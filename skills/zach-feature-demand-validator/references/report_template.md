# 报告模板 — 功能需求验证报告

## YAML 头

```yaml
---

data_sources: Sorftime MCP / review_source_pack / WebSearch

---
```

## 写作规则

1. 区分三种标签：
   - `[数据事实]`
   - `[公开信息]`
   - `[分析推断]`
2. 如果关键词维度未验证，必须明确写出，不能假装完整。
3. Review fallback 场景必须说明 `review_source_pack` 的核查路径。
4. 结论必须引用具体数值，不允许只写“有需求”“讨论很多”。
5. 负面信号必须如实保留。
