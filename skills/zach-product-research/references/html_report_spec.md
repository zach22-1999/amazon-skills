# HTML 精简报告规范

## 必选区块

1. Header
2. KPI 卡片
3. 多维分布表
4. 消费者痛点卡片
5. 供需缺口/机会空白
6. 品牌/竞品格局
7. 进入壁垒评估
8. Go / No-Go评分卡
9. 策略建议
10. Footer

## 样式约束

- 使用 CSS 变量统一颜色
- 容器 `max-width: 960px`
- KPI 用网格布局
- 表格可读、可响应式缩放
- Footer 必须写清数据来源、采集日期、MD 与 XLSX 文件名

## 常见问题

- 不要只给“一段话 + 一个表格”的简版
- 不要出现“约”“少量”“大概”这类模糊词替代数字
- HTML 与 Markdown / Excel 必须同源，不允许三份文件数字对不上
